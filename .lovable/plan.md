
# خطة: تحسين عرض تواريخ الباقة وتطوير نافذة تعديل الباقة

## المشكلة الحالية
1. **تواريخ مكونات الباقة**: السهم يشير لليمين (→) بينما يجب أن يشير لليسار (←) في واجهة RTL، والتاريخ لا يظهر السنة
2. **نافذة تعديل الوثيقة**: صغيرة جداً، وتعرض وثيقة واحدة فقط بدلاً من كل وثائق الباقة

## الحل المقترح

### 1. تصحيح عرض التواريخ في PolicyYearTimeline
- تغيير السهم من → إلى ←
- إظهار التاريخ كاملاً مع السنة (DD/MM/YYYY)

### 2. إعادة تصميم نافذة تعديل الباقة
بدلاً من تعديل وثيقة واحدة، ستعرض النافذة **جميع وثائق الباقة** مع إمكانية تعديل كل واحدة:

| الميزة | القديم | الجديد |
|--------|--------|--------|
| العرض | max-w-md (صغير) | max-w-3xl (كبير وواسع) |
| المحتوى | وثيقة واحدة | جميع وثائق الباقة |
| التصميم | حقول بسيطة | جدول احترافي لكل وثيقة |

## التغييرات التقنية

| الملف | التغيير |
|-------|---------|
| `PolicyYearTimeline.tsx` | تصحيح السهم (→ إلى ←) + إظهار السنة في التاريخ |
| `PackagePolicyEditModal.tsx` | إعادة كتابة كاملة: توسيع + عرض جميع الوثائق |

## التفاصيل التقنية

### 1. تصحيح PolicyYearTimeline (PackageComponentRow)
```tsx
// قبل
{formatDate(policy.start_date).slice(0, 5)} → {formatDate(policy.end_date).slice(0, 5)}

// بعد  
{formatDate(policy.end_date)} ← {formatDate(policy.start_date)}
```

### 2. إعادة كتابة PackagePolicyEditModal

**Props جديدة:**
- `groupId: string` - معرف الباقة
- `onSaved: () => void` - callback بعد الحفظ

**المنطق الجديد:**
1. جلب جميع الوثائق التابعة لـ `group_id`
2. عرض كل وثيقة في بطاقة منفصلة
3. إدارة state لكل وثيقة (التواريخ والسعر)
4. حفظ جميع التغييرات دفعة واحدة

**المكونات:**
- `PackagePolicyCard` - بطاقة لكل وثيقة تحتوي على:
  - أيقونة ولون حسب النوع
  - اسم الشركة (للقراءة فقط)
  - `ArabicDatePicker` × 2 (بدء + انتهاء)
  - `Input` للسعر
- شريط المجموع في الأسفل
- زر حفظ شامل

## النتيجة المتوقعة
- التواريخ تظهر بشكل صحيح مع السنة والسهم للاتجاه الصحيح
- نافذة تعديل احترافية وواسعة
- تعديل جميع وثائق الباقة في مكان واحد
- تجربة مستخدم سلسة ومتناسقة
