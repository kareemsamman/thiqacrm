
# خطة: تحسين تصميم تقرير التجديدات - تنسيق الجدول الموحد

## المشكلة الحالية
التصميم الحالي يستخدم بطاقات منفصلة لكل عميل مما يسبب الكثير من التمرير (Scroll).

## الحل المقترح
تغيير التصميم إلى **جدول واحد موحد** مع:
- صف رئيسي (Header Row) لكل عميل يحتوي على اسمه ومعلوماته
- صفوف فرعية أسفله لكل وثيقة تابعة له
- تصميم مدمج وسريع القراءة

## مميزات التصميم الجديد

| الميزة | الوصف |
|--------|-------|
| **جدول واحد** | كل البيانات في جدول متصل بدون فراغات |
| **صف العميل** | خلفية ملونة مميزة + اسم + هاتف + رقم ملف + ملخص |
| **صفوف الوثائق** | معلومات تفصيلية لكل وثيقة أسفل صف العميل |
| **ترقيم متسلسل** | رقم تسلسلي لكل وثيقة عبر التقرير بالكامل |
| **ألوان الأولوية** | أحمر للعاجل (≤7 أيام)، برتقالي للتحذير (8-14 يوم) |
| **طباعة سهلة** | تصميم مناسب للطباعة بدون قطع البيانات |

## الكود الجديد (ملخص)

بدلاً من `client-card` لكل عميل، سيكون:

```html
<table class="main-table">
  <thead>
    <tr>
      <th>#</th>
      <th>الزبون</th>
      <th>رقم السيارة</th>
      <th>النوع</th>
      <th>الشركة</th>
      <th>تاريخ الانتهاء</th>
      <th>المتبقي</th>
      <th>السعر</th>
    </tr>
  </thead>
  <tbody>
    <!-- صف العميل (Header Row) -->
    <tr class="client-row urgent">
      <td colspan="6">👤 محمد أحمد (0501234567) - #125</td>
      <td>3 وثائق</td>
      <td>₪4,500</td>
    </tr>
    <!-- صفوف الوثائق -->
    <tr class="policy-row">
      <td>1</td>
      <td></td>
      <td>12-345-67</td>
      <td>إلزامي</td>
      <td>حريش</td>
      <td>15/02/2026</td>
      <td class="urgent">3 أيام</td>
      <td>₪1,500</td>
    </tr>
    <!-- ... المزيد من الوثائق -->
  </tbody>
</table>
```
