

# إنشاء نظام تحويلات متعدد مع اختيار التحويلة عند الاتصال

## نظرة عامة

سأقوم بتحويل النظام الحالي من استخدام كلمة مرور واحدة مشتركة إلى:
1. **جدول تحويلات منفصل** - لتخزين عدة تحويلات مع كلمات مرور خاصة لكل واحدة
2. **تحويل MD5 تلقائي** - إدخال كلمة المرور العادية والنظام يحولها
3. **اختيار التحويلة في popup الاتصال** - قائمة منسدلة لاختيار أي تحويلة تستخدم

## التصميم الجديد

### A) جدول التحويلات الجديد `pbx_extensions`

| العمود | النوع | الوصف |
|--------|-------|-------|
| id | UUID | المعرف |
| extension_number | TEXT | رقم التحويلة (مثل 101, 104) |
| extension_name | TEXT | اسم التحويلة (مثل "المكتب الرئيسي") |
| password_plain | TEXT | كلمة المرور العادية |
| password_md5 | TEXT | كلمة المرور مشفرة MD5 (تُحسب تلقائياً) |
| is_default | BOOLEAN | هل هي التحويلة الافتراضية |
| created_at | TIMESTAMP | تاريخ الإنشاء |

## التفاصيل التقنية

### 1. إنشاء جدول قاعدة البيانات

```sql
CREATE TABLE public.pbx_extensions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  extension_number TEXT NOT NULL,
  extension_name TEXT,
  password_plain TEXT NOT NULL,
  password_md5 TEXT NOT NULL,
  is_default BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- RLS للأمان (فقط للمديرين)
ALTER TABLE pbx_extensions ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Admins can manage extensions"
  ON pbx_extensions FOR ALL
  USING (
    EXISTS (
      SELECT 1 FROM user_roles
      WHERE user_id = auth.uid() AND role = 'admin'
    )
  );

-- دالة لحساب MD5 تلقائياً
CREATE OR REPLACE FUNCTION compute_md5_password()
RETURNS TRIGGER AS $$
BEGIN
  NEW.password_md5 := md5(NEW.password_plain);
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_compute_md5
  BEFORE INSERT OR UPDATE ON pbx_extensions
  FOR EACH ROW
  EXECUTE FUNCTION compute_md5_password();
```

### 2. تحديث AuthSettings.tsx

- إضافة قسم لإدارة التحويلات
- عرض جدول التحويلات مع أزرار إضافة/حذف/تعديل
- إدخال كلمة المرور العادية (MD5 يُحسب تلقائياً في قاعدة البيانات)

### 3. تحديث Click2CallDialog.tsx

- جلب قائمة التحويلات المتاحة
- إضافة dropdown لاختيار التحويلة
- التحويلة الافتراضية تكون محددة مسبقاً
- إرسال `extension_id` بدلاً من `extension_number`

### 4. تحديث Edge Function `click2call`

- استقبال `extension_id` من الطلب
- جلب بيانات التحويلة من جدول `pbx_extensions`
- استخدام `password_md5` في الاتصال بـ API

## الفوائد

1. **لا حاجة لحساب MD5 يدوياً** - أدخل كلمة المرور العادية والنظام يتكفل بالباقي
2. **تحويلات متعددة** - يمكن إضافة عدة تحويلات (101, 104, 105...)
3. **مرونة عند الاتصال** - اختر أي تحويلة تريد استخدامها لكل مكالمة
4. **تحويلة افتراضية** - لتسريع العمل اليومي
