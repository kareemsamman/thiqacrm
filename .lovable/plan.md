

# تحسين قواعد المساعد الذكي "ثاقب"

## المشكلة الحالية
- النظام الحالي يعرض حتى 100 نتيجة في الاستعلامات التجميعية و15 في العادية بدون تنبيه واضح
- القواعد (System Prompt) بسيطة وتحتاج تحسين لتكون أكثر ذكاءً وتوجيهاً
- لا يوجد توجيه للمستخدم نحو الصفحات المناسبة عند وجود بيانات كثيرة

## التغييرات المطلوبة

### 1. تحديث System Prompt بقواعد شاملة

تحديث `DEFAULT_SYSTEM_PROMPT` في `supabase/functions/ai-assistant/index.ts` ليشمل:

- **حد العرض**: أقصى 20 نتيجة فقط. إذا كان هناك أكثر، أخبر المستخدم بالعدد الإجمالي ووجهه للصفحة المناسبة (مثلاً: "يوجد 150 عميل، هنا أول 20. يمكنك زيارة صفحة العملاء لرؤية الجميع")
- **اللغة**: دائماً بالعربية، أسلوب مهني وودود ومختصر
- **التوجيه للصفحات**: توجيه المستخدم لصفحة العملاء / الوثائق / السيارات عند الحاجة
- **عدم اختراع بيانات**: لا يفترض أو يخترع بيانات غير موجودة
- **عدم كشف التفاصيل التقنية**: لا يذكر أسماء جداول أو APIs
- **التنسيق**: قوائم منظمة بأرقام، مختصرة وواضحة
- **الذاكرة**: يتذكر سياق المحادثة ويبني عليه
- **الأمان**: لا يعرض بيانات مالية للعاملين (موجود حالياً)

### 2. تحديث حد البيانات في الكود

في `fetchContextData`:
- تغيير `limit` من `15` (عادي) و`100` (تجميعي) إلى `20` كحد أقصى للعرض
- إضافة استعلام `count` لمعرفة العدد الإجمالي الحقيقي
- إرفاق العدد الإجمالي مع البيانات حتى يستطيع الذكاء الاصطناعي إخبار المستخدم (مثلاً: "إجمالي العملاء: 350، هنا أول 20")

### 3. إضافة توجيهات الصفحات

إضافة خريطة للصفحات في System Prompt:
- العملاء → صفحة العملاء
- السيارات → صفحة السيارات  
- الوثائق → صفحة الوثائق
- المدفوعات → صفحة المدفوعات
- شركات التأمين → صفحة الشركات

### الملفات المتأثرة
- `supabase/functions/ai-assistant/index.ts` — تحديث System Prompt + تحديث حدود الاستعلام + إضافة count queries
- إعادة نشر Edge Function
