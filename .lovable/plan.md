## المطلوب
1. **حساب تاريخ الانتهاء تلقائياً**: عند تغيير تاريخ البدء، يتم حساب تاريخ الانتهاء = تاريخ البدء + سنة - يوم
   - مثال: 04/02/2026 ← 03/02/2027
2. **إضافة اختيار السائقين الإضافيين**: إمكانية اختيار سائقين من بيانات العميل أو إضافة سائقين جدد

## التفاصيل

### 1. حساب تاريخ الانتهاء التلقائي

استخدام نفس المنطق الموجود في `PackageBuilderSection.tsx`:

```typescript
// Helper to calculate end date (1 year - 1 day from start)
const calculateEndDate = (startDate: string): string => {
  if (!startDate) return "";
  const start = new Date(startDate);
  const end = new Date(start);
  end.setFullYear(end.getFullYear() + 1);
  end.setDate(end.getDate() - 1);
  return end.toISOString().split("T")[0];
};
```

تعديل `updateEditState` للتعامل مع تغيير startDate:

```typescript
const updateEditState = (policyId: string, field: keyof EditState, value: string) => {
  setEditStates((prev) => {
    const newState = { ...prev[policyId], [field]: value };
    
    // Auto-calculate end date when start date changes
    if (field === "startDate" && value) {
      newState.endDate = calculateEndDate(value);
    }
    
    return { ...prev, [policyId]: newState };
  });
};
```

**جلب البيانات:**
- استخراج `client_id` من أول policy في الباقة
- جلب التابعين الموجودين من `client_children`
- جلب التابعين المرتبطين حالياً من `policy_children`

**UI Component:**
- إعادة استخدام منطق `PolicyChildrenSelector`:
  - Checkboxes للتابعين الموجودين
  - إمكانية إضافة تابعين جدد بنموذج مدمج

**عند الحفظ:**
1. إضافة التابعين الجدد إلى `client_children`
2. ربط جميع التابعين المختارين بالوثيقة الرئيسية (THIRD_FULL) في `policy_children`

## النتيجة المتوقعة
- تغيير تاريخ البدء يحسب تلقائياً تاريخ الانتهاء (سنة - يوم)
- يظهر قسم السائقين الإضافيين مع إمكانية:
  - اختيار من التابعين الموجودين للعميل
  - إضافة تابعين جدد
- عند إرسال الوثيقة للعميل، يظهر السائقين الإضافيين المرتبطين
