

# إضافة قاعدة تسعير لشركة اراضي مقدسة — شامل تحت 60,000

## المشكلة
حالياً عندما تكون قيمة السيارة أقل من 60,000 شيكل مع وثيقة شامل (FULL) لشركة اراضي مقدسة، النظام لا يحسب مبلغ الشامل (لأن الشرط `carValue >= 60000` غير متحقق)، فيصبح المستحق للشركة فقط الخصم (700 شيكل) بدل 1900.

## الحل
إضافة قاعدة `MIN_PRICE` بقيمة **1200** شيكل لشركة اراضي مقدسة.

### لماذا 1200 وليس 1900?
النظام يحسب المستحق للشركة هكذا:
```text
المستحق = جزء الشامل (fullComponent) + جزء الثالث (thirdComponent)
```

حالياً الخصم (DISCOUNT) = 700 شيكل يُستخدم كـ thirdComponent.

لذلك:
```text
1200 (MIN_PRICE) + 700 (DISCOUNT) = 1900 شيكل إجمالي
```

## التغييرات المطلوبة

### إدخال بيانات فقط — لا تغيير في الكود
إضافة صف واحد في جدول `pricing_rules`:

| الحقل | القيمة |
|-------|--------|
| company_id | اراضي مقدسة |
| policy_type_parent | THIRD_FULL |
| rule_type | MIN_PRICE |
| car_type | car |
| age_band | ANY |
| value | 1200 |

### كيف يعمل
- إذا قيمة السيارة >= 60,000 → يُستخدم FULL_PERCENT (1.75% أو 2%)
- إذا قيمة السيارة < 60,000 → يُستخدم MIN_PRICE (1200) + DISCOUNT (700) = **1900 شيكل**
