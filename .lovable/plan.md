
# خطة: إصلاح ربط دفعة الفيزا للعميل موسى هلسة

## ملخص المشكلة

بعد التحقيق في قاعدة البيانات، اكتشفت أن هناك **وثيقتين منفصلتين** للعميل موسى هلسة لنفس السيارة اليوم:

| الوثيقة | النوع | السعر | المدفوع | حالة الدفع |
|---------|-------|-------|---------|------------|
| **باقة** (6c6de223...) | ELZAMI + THIRD_FULL + ROAD_SERVICE | ₪4,836 | ₪1,836 (نقدي) | متبقي ₪3,000 |
| **منفردة** (9f321b0a...) | THIRD_FULL | ₪4,836 | ₪4,836 (₪3,000 فيزا + ₪1,836 نقدي) | مدفوعة |

الدفعة بالفيزا (₪3,000) مُسجلة على الوثيقة **المنفردة** بينما العميل يريد رؤيتها على **الباقة**.

---

## الحل المقترح

### الجزء 1: نقل دفعة الفيزا إلى الباقة

سيتم تحديث `policy_id` لدفعة الفيزا لتشير إلى إحدى وثائق الباقة:

```sql
UPDATE policy_payments
SET policy_id = '62309a9a-142f-48f2-8744-764dc08f94b6'  -- THIRD_FULL في الباقة
WHERE id = '681c996a-fcdf-4847-9925-cc5c5bf20259';
```

### الجزء 2: حذف الدفعة النقدية المكررة

هناك دفعة نقدية ₪1,836 على الوثيقة المنفردة يجب حذفها لمنع الدفع المزدوج:

```sql
DELETE FROM policy_payments
WHERE id = 'fc986839-dde1-4365-ba4c-4629f85d6a7a';
```

### الجزء 3: حذف الوثيقة المنفردة (soft delete)

لمنع ظهورها في التقارير وتكرار المبالغ:

```sql
UPDATE policies
SET deleted_at = now()
WHERE id = '9f321b0a-3911-49dc-a207-2af32a4e7b16';
```

## النتيجة المتوقعة بعد الإصلاح

| الباقة | السعر | المدفوع | الحالة |
|--------|-------|---------|--------|
| ELZAMI | ₪1,836 | ₪1,836 | ✓ مدفوع |
| THIRD_FULL | ₪2,500 | ₪3,000 (فيزا) | ✓ مدفوع |
| ROAD_SERVICE | ₪500 | - | ✓ مدفوع (من الزيادة) |
| **المجموع** | **₪4,836** | **₪4,836** | **مدفوع بالكامل** |

## تنفيذ التغييرات

سأقوم بتطبيق هذه التغييرات مباشرة على قاعدة البيانات، ثم ستحتاج إلى تحديث صفحة العميل لرؤية الأرقام الصحيحة.

---

## ملاحظة مهمة للمستقبل

لقد تم إصلاح الكود في `tranzila-init` سابقاً لإضافة `myid` إلى طلب Tranzila. هذا يمنع تكرار هذه المشكلة في المستقبل حيث ستتمكن الـ callback من تحديث الدفعة الصحيحة تلقائياً.
